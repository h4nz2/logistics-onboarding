You are designing backend for a logistics platform, that helps ecommerce companies to efficiently restock their products. This repository is just an exercise focus on implementing the onboarding part of this software. Do not implement any other parts that are not needed for onboarding.

Requirements and description of the architecture and all decision can be found in `docs/decisions.md`. The description should be general and should not go too much into detail. It should focus on general decisions made.

Suggestions for improvements can be found in `docs/suggestions.md`

If you are not sure about information or what you should deliver, ask additional questions to better understand the requirements.

Whenever you complete any task, do the following:

1. Verify that requirements are met
2. Verify that description of architecture and all decisions is up to date
3. Try to come up with suggestions for improving the onboarding (user experience, technical improvements etc.). Do not modify the suggestions file directly, just print the suggestions.
